    TinyGoogle : A client-server based parallel Search Engine

    TinyGoogle
  • The project involves create a client-server based parallel search engine for document indexing and word searching.
  • Implemented a client-server setup using TCP sockets with a single master server and dynamically selected worker nodes.
  • Server sends Map-Reduce request to worker nodes for indexing and Search request for searching. Selectiton is based on availability. The method is compared with Hadoop Map-Reduce framework
  • The response time for searching reduces significantly with increasing parallel worker nodes (Over 70% reduction with 5 nodes vs 2 nodes for 5 words)
  • Key technologies: Python, Threading, Hadoop
  • Domain : Distributed OS, Parallel Processing
